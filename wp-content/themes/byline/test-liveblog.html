LiveBlog Functionality Test

Test Instructions

1. Go to WordPress Admin → LiveBlog → Edit the existing LiveBlog post

2. Try clicking "Add New Update" button

3. Check if TinyMCE editor loads without errors

4. Check browser console for any JavaScript errors

Expected Behavior

  • ✅ "Add New Update" button should create a new block
  • ✅ TinyMCE editor should load without "print plugin" errors
  • ✅ Content should sync between editor and hidden textarea
  • ✅ Publish/Draft buttons should work independently

Debugging Steps

  1. Open browser developer tools (F12)
  2. Go to Console tab
  3. Look for any error messages
  4. Check if "addLiveBlogBlock called" appears when clicking the button
  5. Check if "TinyMCE editor initialized successfully" appears

Common Issues & Solutions

  • TinyMCE not loading: Check if wp_enqueue_editor() is called
  • Add button not working: Check for JavaScript errors in console
  • Content not saving: Check AJAX nonce and permissions
